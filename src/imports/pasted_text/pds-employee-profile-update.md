Update all **Employee Profile pages across the HRIS** so that they follow the **same Personal Data Sheet (PDS – CS Form 212 Revised 2025) layout and UI structure**.

This applies to the following pages:

• Employee “My Profile” page (Employee view)
• Employee Profile panel in the HR Officer Employee Directory
• Employee Profile panel in the Department Head Employee List

All pages must display **the same PDS sections, layout structure, and UI components**.

Do not change the HRIS design system such as colors, typography, sidebar layout, header design, card styles, spacing, or icons.

---

GLOBAL UI STRUCTURE

Convert the PDS form into a **modern profile interface using sections or cards**.

Each PDS section should appear as a **separate card or expandable panel**.

All views must follow the same **section order**.

---

At the top of the page display a profile summary card containing:

Employee Photo
Full Name
Employee ID
Position Title
Department
Employment Status
Date Hired

Email Address

Telephone Number

VIEW PERMISSIONS

Employee View
Employees may edit limited personal details such as:

Contact Information
Address
Phone Number

HR Officer View
HR Officers may view and edit the full PDS.

Department Head View
Department Heads may only **view the PDS record** and cannot edit official HR information.

GOAL

Ensure all employee profile pages across the HRIS use the **same PDS-based layout and UI design**, creating a standardized employee record interface consistent with **CS Form 212 (Personal Data Sheet)**.
