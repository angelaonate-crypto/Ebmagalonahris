Update the **HR Staff Settings page** so that it follows the same structure and layout style used in the **Employee Settings interface**.

Do not change the HRIS design system, colors, typography, spacing, sidebar layout, or header components. Only reorganize how the settings are structured.

---

SETTINGS STRUCTURE

Instead of displaying all settings in a single page, divide the HR Staff settings into **separate sections**, similar to how the Employee Settings page is organized.

Each settings category should be separated and accessible individually.

---

SETTINGS NAVIGATION

Create a settings navigation panel or tabs for:

Profile Settings
Account Settings
Security Settings
Notification Settings
System Preferences

Each category should open its own section.

---

PROFILE SETTINGS

Fields may include:

Full Name
Email Address
Contact Number
Profile Photo

Allow HR Staff to update their personal profile information.

---

ACCOUNT SETTINGS

Fields may include:

Username
Change Password
Two-Factor Authentication toggle

---

SECURITY SETTINGS

Display options such as:

Enable Facial Verification
Session Timeout Settings
Login Activity History

HR Staff can view their login activity but cannot modify system-wide security settings.

---

NOTIFICATION SETTINGS

Allow HR Staff to manage notification preferences such as:

System notifications
Leave verification alerts
Applicant submission notifications
Training notifications

Use toggle switches for each notification type.

---

SYSTEM PREFERENCES

Allow configuration of:

Language
Theme
Time zone
Date format

These preferences apply only to the HR Staff user's interface.

---

PERMISSIONS

HR Staff settings should only affect **their own account** and should not modify global HRIS configurations.

Administrative system settings remain under the **HR Officer role**.

---

GOAL

Ensure the **HR Staff Settings interface follows the same structured layout used in Employee Settings**, where each category is clearly separated and easier to manage while respecting HR Staff permission levels.
